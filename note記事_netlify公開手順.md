## STEP 1：GitHub Pagesは商用NG！？急遽Netlifyへ乗り換えた理由

<!-- 📸 画像⑲（Netlifyトップページ）をここに挿入 -->

実はこのサイト、最初は**GitHub Pages**で公開していました。

<!-- 📸 画像④（GitHub Pages設定画面）をここに挿入 ※tsukushihubをぼかす -->

「無料でHP公開できる！」と喜んでいたのですが、調べてみると…

> ⚠️ **GitHub Pagesは商用利用ができません**
> 集客・販売・広告掲載を目的としたサイトへの使用は利用規約で禁止されています。

知らずにやっていたので焦りました。急遽、**Netlify**に乗り換えることにしました。

**Netlifyのメリット：**
- 無料でHTTPS（鍵マーク）対応
- **商用利用OK**
- GitHubと連携すれば、コード更新→自動で反映
- 独自ドメインも設定可能

まずは netlify.com でアカウントを作成します。GitHubアカウントでのサインアップが後々便利です。

## STEP 2：まずはGitHubにコードを送信しましょう

NetlifyはGitHubと連携して動きます。まずGitHubにファイルを送ります。

<!-- 📸 画像①（GitHub Sign in画面）をここに挿入 -->
<!-- 📸 画像②（Git Credential Manager認証画面）をここに挿入 ※tsukushihubをぼかす -->
<!-- 📸 画像③（GitHubリポジトリ一覧）をここに挿入 ※アイコンをぼかす -->

PowerShell（またはコマンドプロンプト）を開いて、以下を順番に実行してください。

```powershell
# ① プロジェクトフォルダに移動
cd プロジェクトのフォルダパス

# ② 変更したファイルをすべてステージング
git add .

# ③ 変更内容にメモをつけて保存
git commit -m "商用利用のための機能追加と画像パスの修正"

# ④ GitHubに送信
git push origin main
```

⚠️ **詰まりポイント：mainでエラーが出る場合**

GitHubのデフォルトブランチ名は環境によって `main` または `master` が異なります。エラーが出たら以下を試してください。

```powershell
git push origin master
```

5. デプロイ内容を確認して「**Deploy ly-hair-beautysalon to Netlify**」をクリック

## STEP 4：URLをお店の名前に変えて、スッキリさせましょう

Netlifyが自動でつけるURLは `cheerful-starburst-〇〇〇〇.netlify.app` のようなランダムな名前です。これをお店の名前に変更できます。

<!-- 📸 画像⑬（変更前のURL：cheerful-starburst）をここに挿入 -->

1. Netlifyの管理画面でサイトを開く
2. 左サイドバーの「**Project configuration**」をクリック
3. 「**General project settings**」の画面を下にスクロール
4. 「**Project name**」の横にある「**Change project name**」をクリック
5. 好きな名前（例：`ly-hair`）を入力して保存

<!-- 📸 画像⑭（変更後のURL：ly-hair.netlify.app）をここに挿入 -->
<!-- 📸 画像㉔（Project configuration画面）をここに挿入 ※Owner・Project IDをぼかす -->

## STEP 5：プロの商用サイトとして、中身をさらに整えます

公開できたら終わり、ではありません。商用利用するための重要なステップがまだ残っています。AntigravityのAIと相談しながら実装した内容が、こちらです。

### ① SEO・SNS対策（OGP & JSON-LD）
検索エンジンに店舗情報を正しく伝えるための設定です。SNSでシェアされたときにサムネイルや説明文が表示されるようになります。

### ② プライバシーポリシーページの追加
個人情報を扱う商用サイトには必須です。問い合わせフォームがある場合は特に重要になります。フッターにリンクを設置しました。

### ③ モバイル最適化
- 電話番号をタップで即発信できるように設定
- ハンバーガーメニューのアニメーションを改善

### ④ 信頼性の向上
- Instagramへのリンクを設置
- フッターデザインの微調整
