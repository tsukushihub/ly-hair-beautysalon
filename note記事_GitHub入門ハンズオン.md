# GitHubって何？アカウント作成からファイルアップまで【初心者ハンズオン・Windows版】

## はじめに

「GitHub（ギットハブ）って名前は聞いたことあるけど、何をするものかわからない」

そんな方に向けて、**実際に手を動かしながら学べる**ように書きました。難しい言葉はなるべく使わず、画面の操作を一つひとつ丁寧に説明します。

この記事を読み終わると、GitHubにファイルをアップロードするところまでできるようになります。その先でNetlifyと連携すれば、**無料でHPを公開することも可能**です。

**この記事でわかること：**
- GitHubとは何か・何のために使うか
- アカウントの作り方
- Gitのインストール方法（Windows）
- ファイルをGitHubにアップロードするまでの手順

---

## GitHubって、結局何なの？

一言で言うと、**ファイルをインターネット上に保存・管理できる場所**です。

普段使うGoogleドライブやDropboxに似ていますが、GitHubは特に**プログラムやウェブサイトのファイル管理**に特化しています。

**GitHubを使うと何がいいの？**
- ファイルの変更履歴が全部残る（「あの時の状態に戻したい」が叶う）
- NetlifyなどのサービスとつなぐとHPを自動で公開・更新できる
- 無料で使える

> 💡 「Git（ギット）」はファイルの変更を記録する仕組みの名前。「GitHub」はそれをインターネット上で使えるようにしたサービスです。

## STEP 1：GitHubのアカウントを作る

まずは github.com にアクセスします。

<!-- 📸 GitHubトップページのスクリーンショットをここに挿入 -->

1. 「**Sign up**」をクリック
2. メールアドレス・パスワード・ユーザー名を入力
3. 認証メールが届くので、コードを入力して完了

> 💡 ユーザー名はURLに表示されます。本名でなくてもOKですが、あとから変更しにくいので慎重に決めましょう。

---

## STEP 2：Gitをパソコンにインストールする

GitHubを使うには、まずパソコンに「Git」というソフトをインストールする必要があります。

<!-- 📸 Git公式サイトのスクリーンショットをここに挿入 -->

1. git-scm.com にアクセス
2. 「**Download for Windows**」をクリック
3. ダウンロードされたファイルをダブルクリックして実行
4. 設定はすべてデフォルトのまま「Next」を押し続けてインストール完了

**インストールできたか確認する方法：**

PowerShell（またはコマンドプロンプト）を開いて以下を入力します。

```powershell
git --version
```

`git version 2.○○.○` のように表示されれば成功です！

## STEP 3：GitHubに「リポジトリ」を作る

リポジトリとは、**ファイルを入れる専用のフォルダ**のようなものです。プロジェクトごとに1つ作るのが基本です。

1. GitHubにログインして右上の「**+**」→「**New repository**」をクリック
2. Repository name（名前）を入力（例：`my-website`）
3. 「**Public**」か「**Private**」を選ぶ（公開・非公開）
4. 「**Add a README file**」にチェックを入れる
5. 「**Create repository**」をクリック

<!-- 📸 リポジトリ作成画面のスクリーンショットをここに挿入 -->

これでGitHub上に保存場所ができました！

---

## STEP 4：パソコンのフォルダとGitHubを繋げる

次に、自分のパソコン上のフォルダとGitHubを連携させます。

PowerShellを開いて、以下を順番に実行してください。

```powershell
# ① 名前とメールアドレスを登録する（初回だけ）
git config --global user.name "あなたの名前"
git config --global user.email "メールアドレス"

# ② プロジェクトのフォルダに移動する
cd フォルダのパス

# ③ このフォルダをGitで管理する準備をする
git init

# ④ GitHubのリポジトリと繋げる
git remote add origin https://github.com/ユーザー名/リポジトリ名.git
```

> 💡 フォルダのパスはエクスプローラーでフォルダを開き、アドレスバーをクリックするとコピーできます。

---

## STEP 5：ファイルをGitHubにアップロードする

いよいよ最後のステップです。「add → commit → push」の3ステップを覚えれば大丈夫です。

```powershell
# ① 変更したファイルをステージング（送る準備）
git add .

# ② 変更内容にメモをつけて保存
git commit -m "最初のアップロード"

# ③ GitHubに送信
git push origin main
```

<!-- 📸 pushが成功した画面をここに挿入 -->

GitHubのリポジトリページを開いて、ファイルが表示されていれば成功です！

**「add・commit・push」ってどういう意味？**
- **add**：「このファイルを送る準備するよ」という宣言
- **commit**：「この変更を記録するよ」という保存
- **push**：「GitHubに送信するよ」というアップロード

この3つはセットで覚えておきましょう。
